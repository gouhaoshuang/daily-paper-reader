## 摘要
随着生成式AI能力的扩展，AI驱动的虚拟世界面临着日益增长的架构挑战。用户通过世界内界面以多模态方式进行交互，但其请求需要根本不同的AI后端模型和计算资源。将这些能力直接嵌入虚拟世界系统会降低可扩展性，使维护复杂化，并限制协调分布在边缘和云基础设施上的服务的能力。本文提出了一种基于SLM的智能体编排网关，这是一种轻量级的运行时协调机制，通过意图驱动的服务路由将虚拟世界客户端与异构AI后端解耦。边缘部署的SLM对每个用户提示的语义意图进行分类，可配置的服务注册表验证并解析路由决策，然后透明地调用选定的后端，从而能够在虚拟世界中引入新的AI能力而无需修改客户端应用程序。该网关在InterwovenXR虚拟博物馆测试平台上实现和评估。评估表明，紧凑型SLM可以在边缘硬件上作为可靠的意图路由器，并且任务特定的微调可以将数十亿参数以下的模型转化为实用的低延迟路由器。一种分层配置，将微调后的数十亿参数以下模型作为路由器，与更大的SLM用于对话响应生成相结合，被证明可以在中端边缘硬件上部署，并且比将两个职责委托给单个模型更高效。研究结果表明，SLM可以支持虚拟世界中实用的AI服务编排，并且该工作贡献了一个经过评估的架构，用于可扩展、可扩展且边缘支持的AI交互，使虚拟代理人成为分布式生成式AI服务的访问点。

## Abstract
As generative AI capabilities expand, AI-driven virtual worlds face a growing architectural challenge. Users interact through in-world interfaces in multimodal ways, yet their requests demand fundamentally different AI backend models and computational resources. Embedding these capabilities directly into virtual world systems reduces extensibility, complicates maintenance, and limits the ability to coordinate services distributed across edge and cloud infrastructure. This paper presents an SLM-based Agent Orchestration Gateway, a lightweight runtime coordination mechanism that decouples a virtual world client from heterogeneous AI backends through intent-driven service routing. An edge-deployed SLM classifies the semantic intent of each user prompt, a configurable service registry validates and resolves the routing decision, and the selected backend is invoked transparently, enabling new AI capabilities to be introduced in the virtual world without modifying the client application. The gateway is implemented and evaluated within the InterwovenXR virtual museum testbed. The evaluation shows that compact SLMs can serve as reliable intent routers on edge hardware, and that task-specific fine-tuning can transform sub-billion-parameter models into practical, low-latency routers. A layered configuration pairing a fine-tuned sub billion-parameter model as router with a larger SLM for conversational response generation is shown to be deployable on mid-range edge hardware and more efficient than delegating both responsibilities to a single model. The findings show that SLMs can support practical AI service orchestration in virtual worlds and the work contributes an evaluated architecture for scalable, extensible, and edge-supported AI interaction, enabling virtual agents become access points to distributed generative AI services.

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：提出**SLM-based Agent Orchestration Gateway**，一个位于虚拟世界客户端与多个AI后端之间的运行时协调层。它利用边缘部署的小型语言模型（SLM）对用户自然语言提示进行语义意图分类，然后根据分类结果将请求路由到对应的后端服务，并封装响应返回给客户端。
- **关键技术细节**：
  - **架构组件**：包括虚拟世界客户端、编排网关、意图路由器（SLM）、服务注册表与验证层、后端服务执行层。
  - **意图路由**：SLM接收动态生成的路由提示（包含服务注册表中的服务描述），输出结构化JSON决策（选中路由、意图标签、置信度、标准化查询）。
  - **服务注册表**：维护可用服务的路由名称、类型、端点、启用状态和描述，用于构造路由选项并验证路由决策（无效路由回退或请求澄清）。
  - **后端执行**：支持OpenAI兼容聊天、普通HTTP、JSON HTTP等模式；后端服务可部署在边缘网络或云端。
  - **分层配置**：使用一个微调后的亚十亿参数SLM（如Qwen2.5-0.5B-FT）作为第一级快速意图路由器，仅当路由到“对话”服务时，由另一个较大的紧凑SLM（如Qwen2.5-1.5B）生成本地对话响应，避免远程调用。
  - **部署**：网关基于Python FastAPI实现，SLM通过llama.cpp加载，全部运行在NVIDIA Jetson Orin NX 8GB边缘设备上。

## 3. 实验设计
- **数据集/场景**：

  - **Stage1**：500条自然语言提示，均匀分布在5个路由类别（各100条）：对话、语言适配、领域专家（马赛克解读）、知识检索（教堂史实）、3D资产生成。提示设计避免关键词匹配，要求语义理解。
  - **Stage2**：100条对话提示，从虚拟世界客户端直接提交，测量端到端延迟。
- **Benchmark**：
  - 使用Qwen2.5-14B-Instruct-Q5作为高容量参考模型（非候选路由器，仅用于验证数据集可区分性）。
- **对比方法**：
  - **候选路由器**：多个紧凑SLM（Qwen2.5-1.5B、SmolLM2-1.7B、Qwen2.5-0.5B、SmolLM2-360M等）及其微调版本。
  - **失败案例**：TinyLlama-1.1B、StableLM2-1.6B因无法稳定输出有效JSON被剔除。
  - **消融**：比较基模型 vs. 微调模型（SmolLM2-360M-FT、Qwen2.5-0.5B-FT）；比较分层配置（路由+对话分离） vs. 单一模型（同时做路由和对话）。

## 6. 主要结论与发现
- **RQ1**：紧凑SLM可作为虚拟世界中的意图路由机制。
  - Qwen2.5-1.5B路由准确率达94.6%，零无效输出；SmolLM2-1.7B为85.4%。
  - 亚十亿参数基模型（如Qwen2.5-0.5B）准确率仅24.8%，但微调后提升至83.0%（增加58.2个百分点），且延迟仅从329.3ms微增至392.0ms。
  - 路由错误多发生在语义相邻类别（如知识检索 vs. 领域专家），提示服务边界设计的重要性。
- **RQ2**：分层配置比单模型更高效。
  - 所有分层配对（路由+对话两模型同时加载于边缘设备）的端到端平均延迟均低于1.6秒，中位数低于1.5秒，满足2秒交互阈值。
  - 例如Qwen2.5-0.5B-FT + Qwen2.5-1.5B平均1454ms，优于单独使用Qwen2.5-1.5B作为路由（延迟1777ms）加上后续对话生成。
  - 分离职责减少了第一级决策的计算开销，同时保留了较大模型生成对话的能力。
  - **最佳组合**：Qwen2.5-0.5B-FT（路由）+ Qwen2.5-1.5B（对话）在准确率和延迟间取得平衡。

## 7. 优点

- **可复现性**：详细说明了数据集构造、微调配置、部署环境（llama.cpp、NVIDIA Jetson等），便于复现。
