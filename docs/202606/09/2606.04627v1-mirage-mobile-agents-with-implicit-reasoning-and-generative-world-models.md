## 摘要
移动智能体越来越多地需要从截图和语言目标中操作日常应用，其中可靠控制需要推理屏幕功能、多步导航和未来状态变化。然而，许多智能体将这种计算外部化为冗长的文本思维链，这减慢了交互速度，增加了监督成本，并使部署复杂化。我们提出了MIRAGE，一个从可见文本推理轨迹中学习连续潜在推理表征的框架。MIRAGE将显式推理转化为紧凑的隐藏状态，使智能体能够在内部推理，而无需解码冗长的推理过程。它还包含一个生成式世界模型目标：潜在推理向量与未来截图对齐，鼓励智能体在行动前预测即将出现的界面状态。这使隐藏计算既成为压缩的思想表征，又成为环境动态的前瞻模型。在推理时，MIRAGE在连续潜在空间中进行推理，减少了令牌生成，同时提高了执行效率。在AndroidWorld上，MIRAGE与显式思维链监督微调（4B消融实验）相比，解码令牌预算降低3-5倍，并将可比的指令调优基线提高了10.2个点；在AndroidControl上，它在减少75%以上令牌生成的同时改进了动作定位。

## Abstract
Mobile agents are increasingly expected to operate everyday applications from screenshots and language goals, where reliable control requires reasoning over screen affordances, multi-step navigation, and future state changes. However, many agents externalize this computation as long textual chains of thought, which slows interaction, increases supervision cost, and complicates deployment. We introduce MIRAGE, a framework that learns continuous latent reasoning representations from visible textual reasoning traces. MIRAGE transfers explicit reasoning into compact hidden states, enabling the agent to reason internally without decoding long rationales. It also incorporates a generative world-model objective: latent reasoning vectors are aligned with future screenshots, encouraging the agent to anticipate upcoming interface states before acting. This turns hidden computation into both a compressed thought representation and a forward-looking model of environment dynamics. At inference time, MIRAGE reasons in continuous latent space, reducing token generation while improving execution efficiency. On AndroidWorld, MIRAGE matches explicit chain-of-thought supervised fine-tuning in the 4B ablation with a 3-5x lower decoded-token budget and improves a comparable instruction-tuned baseline by 10.2 points; on AndroidControl, it improves action grounding while generating over 75% fewer tokens.

## 1. 核心问题与整体含义（研究动机和背景）

**研究动机**  
- 现有移动GUI智能体（如UI-TARS、OS-ATLAS等）依赖**显式思维链（CoT）**进行多步推理：先输出长文本的观察、理由和预测，再输出动作。这导致解码时间长、上下文消耗大、监督成本高，难以高效部署。  
- 核心矛盾在于：推理能力与部署效率之间的权衡。显式CoT提供可解释性并提升决策质量，但每个额外token都会拖慢交互、增加延迟。

**整体含义**  
- MIRAGE旨在将推理**内化到模型内部**，即**隐式推理**：使用连续潜在变量替代显式文本思维链，在动作生成前进行内部计算，同时通过学习世界模型（预测下一帧视觉特征）让潜在表示具备前瞻性。  
- 这允许智能体在保持显式CoT推理优势的同时，大幅减少输出token数量，从而降低延迟，提升交互效率。

## 2. 方法论

### 核心思想
将移动GUI控制建模为**潜在思维生成问题**：不再解码显式的文本推理轨迹，而是让模型在动作输出前产生一组连续的潜在向量（latent slots），这些向量内部编码了观察、理由和未来状态预测信息。通过两阶段课程训练，逐步从显式CoT过渡到隐式潜在推理，并利用**世界模型对齐**（预测下一帧截图特征）进一步规范潜在表示。

### 关键技术细节

#### 2.1 问题形式化
- 传统显式CoT：  
  `pθ(τt, at | ot) = Π pθ(τt^(i) | τt^(<i), ot) · Π pθ(at^(j) | τt, at^(<j), ot)`  
  其中τt是显式推理文本。
- MIRAGE潜在推理：  
  `pθ(at | ot) = ∫ pθ(at | zt, ot) qθ(zt | ot) dzt`  
  zt是连续潜在变量，不在词汇表中，无需解码为文本。

#### 2.2 两阶段训练流程

**阶段一：显式CoT监督微调**  
- 使用结构化思维格式：`<THOUGHT> [observation] [rationale] [predict] <ACTION_DESC> ... <ACTION> ...`  
- 标准下一个token预测训练，让模型掌握行动格式化与推理模式。

**阶段二：潜在CoT与APLR细化**  
- 将整个`<THOUGHT>`块替换为N个潜在槽（latent slots），每个槽由一个学习到的初始化嵌入elat占据。  
- 引入**近似并行潜在细化（APLR）**：  
  - 传统Coconut风格需要串行更新每个槽（N次前向传播）。  
  - APLR采用**Jacobi风格并行更新**：所有槽在同一轮中使用上一轮的值同时更新，公式：  
    `z^(k+1)_i = Gi( z^(k)_1, ..., z^(k)_{i-1}; c )`，i=1..N，k=0..K-1  
  - 经K轮并行后，前K个槽**精确等于**串行解，尾部槽误差有界（严格三角系统性质）。  
  - 默认K=3，大幅降低训练成本。  
- 最终动作交叉熵损失： `Lce = -Σ log pθ(at^(j) | z^(K)_t, at^(<j), ot)`。

#### 2.3 世界模型对齐（Q-Former下一帧特征对齐）
- 利用训练序列中非终止步骤的**下一帧截图xt+1**作为监督。  
- 构建**Q-Former头**：  
  - 输入：最后一轮APLR后的潜在槽隐藏状态Ct（维度L×H） + 基于网格坐标的可学习位置查询qj。  
  - 输出：预测的下一帧视觉特征 ˆVt+1（每个patch一个向量）。  
  - 目标：使用VLM自身冻结的视觉编码器提取目标特征Vt+1（stop-gradient），计算**掩码逐patch余弦距离**损失：  
    `Lwm = (1/|M|) Σ (1 - ⟨ˆv_j, v_j⟩ / (||ˆv_j|| ||v_j||))`。  
- 世界模型强制潜在槽编码环境动态信息，补偿APLR尾部误差对动作预测的潜在影响。  
- 联合损失：`L = λLce + (1-λ)Lwm`，默认λ=0.8。

#### 2.4 推理阶段
- 去掉Q-Former头，只保留VLM+潜在槽，输入当前截图和历史，通过APLR（但只做一次前向即可）得到潜在表示，然后直接解码动作token，不输出任何文本推理。

## 6. 主要结论与发现

1. **隐式潜在推理可匹配显式CoT性能**：在AndroidWorld上，MIRAGE-4B达到52.6% SR，与explicit CoT SFT相同，且生成的token数减少3-5倍。  
2. **显著减少输出token**：在AndroidControl低指令子集，MIRAGE-4B每步仅18.92 token，比Qwen3-VL-4B-Instruct的115.67减少83.6%；高指令也类似减少75%以上。  
3. **端到端任务成功率提升**：在AndroidWorld上，MIRAGE-8B以57.8% SR超过所有同尺寸专用方法，且平均每步token仅27（为最少的）。  
4. **世界模型补偿尾部误差**：APLR单独使SR下降至48.2，添加Q-Former世界模型后恢复至52.6，验证了下一帧特征对齐的有效性。  
5. **潜槽数量与APLR轮数敏感**：9槽优于3槽；3轮APLR优于2轮（8B模型从46.6提升至57.8）。  
6. **推理延迟最低**：MIRAGE-4B首尾token延迟1.81s，远低于其他方法（2.95-4.52s）。

## 7. 优点

- **创新性方法**：将连续潜在思维链（CoT）引入移动GUI领域，结合并行动态（APLR）大大降低训练成本，同时利用世界模型对齐给予额外监督。
