## 摘要
在边缘服务器上部署轻量级大语言模型（LLM）智能体可以降低延迟，并使智能体服务更贴近用户，但资源受限的边缘模型往往难以处理需要持久记忆、子目标跟踪和反思的长期任务。部署后微调边缘模型成本高昂且难以在异构节点上扩展，而纯本地记忆则使智能体拥有孤立的经验并导致提示上下文不断增长。我们提出\textsc{CoMIC}，一种无需参数更新的云边协作记忆与洞察循环框架。\textsc{CoMIC}遵循“集中式反思，分散式执行”设计：边缘智能体使用面向子目标的层次化记忆和对相关历史的选择性重新展开进行本地执行，而云端LLM评论器异步评估完成轨迹、过滤可重用经验，并聚合以语义子目标标识符为键的跨智能体指导。在涵盖符号规划和文本交互的五项长期智能体任务中，\textsc{CoMIC}提升了弱边缘智能体的进度率与行动基础，并在不更新模型参数的情况下获得了任务相关的成功率提升。

## Abstract
Deploying lightweight Large Language Model (LLM) agents on edge servers can reduce latency and move agentic services closer to users, but resource-constrained edge models often struggle with long-horizon tasks that require persistent memory, subgoal tracking, and reflection. Fine-tuning edge models after deployment is costly and difficult to scale across heterogeneous nodes, while purely local memory leaves agents with isolated experience and growing prompt context. We propose \textsc{CoMIC}, a parameter-update-free cloud-edge framework for Collaborative Memory and Insights Circulation. \textsc{CoMIC} follows a \textit{Centralized Reflection, Decentralized Execution} design: edge agents execute locally using subgoal-oriented hierarchical memory and selective re-expansion of relevant histories, while a cloud-side LLM critic asynchronously evaluates completed trajectories, filters reusable experience, and aggregates cross-agent guidance keyed by semantic subgoal identifiers. Across five long-horizon agent tasks spanning symbolic planning and text interaction, \textsc{CoMIC} improves progress rate and action grounding for weak edge agents and yields task-dependent success-rate gains without updating model parameters.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
**“集中式反思，分散式执行”（Centralized Reflection, Decentralized Execution）**：边缘节点负责在线、实时执行任务，云端LLM作为批评者（critic）异步处理已完成轨迹，过滤可重用经验，并基于语义子目标标识符聚合跨代理指导，最后将精炼后的全局指导（Global Guidance）返回给边缘节点，融入后续决策。

### 关键技术细节
1. **边缘端框架**：
   - **子目标驱动的分阶段执行**：将长期任务分解为子目标片段（subgoal episode），每个子目标记录轨迹 $\hat{\tau}_{it} = (g, g_i^t, o_i^t, a_i^t, r_i^t)$。
   - **层次化记忆与选择性重扩**：当前子目标维护详细的动作-观察对，已完成的子目标压缩为摘要；必要时根据索引集 $I$ 动态重新展开相关历史 $H_t(I)$。
   - **异步轨迹上传**：不阻塞本地执行，保证实时性。

2. **云端批评者框架**：
   - **单轨迹规范化与评估**：将上传轨迹与规范化特征结合，生成串行化记录 $x_{it}$；通过缓存避免重复调用LLM。
   - **门控经验蒸馏**：评估输出含摘要、洞察、建议和自报告置信度 $s_{\text{self}}$；引入准入分数 $s_{\text{adm}}(x_{it}) = \text{clip}_{[0,1]}(s_{\text{self}} - \lambda_{\text{retry}} I_{\text{retry}} - \lambda_{\text{short}} I_{\text{short}})$，控制低质量经验入库。
   - **记忆组织**：云端含短期缓冲（$M_{\text{STM}}$）和知识库（经验记忆 $M_{\text{exp}}$ + 指导记忆 $M = M_{\text{manual}} \cup M_{\text{global}}$）。经验集用阈值 $\gamma_{\text{kb}}$ 筛选。

3. **云边协作机制**：
   - **跨边缘聚合**：按语义子目标标识符 $u$ 分组经验 $G(u)$，聚合生成全局指导 $\tilde{G}(u)$，并发给匹配的边缘节点。
   - **协作上下文组装**：边缘代理在发起新子目标时，用函数 $\mathcal{A}$ 组装完整提示 $P_{t+1}$，包含全局任务、当前子目标、观察、重构历史和云端指导。
