## 摘要
在边缘设备上部署大型语言模型（LLM）可以为不同用户提供个性化的LLM智能体。日益多样化的个性化智能体为点对点（P2P）协作提供了独特机遇，其中每个用户可以将超出本地智能体专业能力的任务委托给更适合特定查询的远程智能体。本文介绍了PPAI，首个个性化LLM智能体互操作性系统，它使用户能够基于智能体的专业化相互协作。然而，与现有P2P系统相比，智能体池的不断变化及其可互换的能力在将查询匹配到智能体以及平衡负载方面带来了新的挑战。因此，我们提出了一种基于原型的可扩展查询-智能体对评分机制，以在存在节点流失的P2P网络中识别合适的智能体。此外，我们提出了一种多智能体互操作性贝叶斯博弈，以在远程智能体负载变化过快而无法观测时平衡本地需求与全局效率。最后，我们实现了PPAI的原型，并证明它在维持负载平衡的同时显著扩展了可执行任务的范围。平均而言，它在多个任务上实现了高达7.96%的准确率提升，同时相比基线延迟降低了16.34%。

## Abstract
Deploying large language model (LLM) on edge device enables personalized LLM agents for various users. The growing availability of diverse personalized agents presents a unique opportunity for peer-to-peer (P2P) collaboration, wherein each user can delegate tasks beyond the local agent's expertise to remote agents more suited for the specific query. This paper introduces PPAI, the first personalized LLM agent interoperability system, which enables users to collaborate with each other based on agent specialization. However, the ever-changing pool of agents and their interchangeable capacity introduce new challenges when it comes to matching queries to agents and balancing loads, compared with existing P2P systems. Therefore, we propose a scalable query-agent pair scoring mechanism based on prototypes to identify suitable agents within a P2P network with churn. Moreover, we propose a multi-agent interoperability Bayesian game to balance local demand and global efficiency, when changes in remote agent load occur too quickly to be observed. Finally, we implement a prototype of PPAI and demonstrate that it substantially broadens the range of tasks that could be carried out while maintaining load balance. On average, it achieves an accuracy improvement of up to 7.96% across multiple tasks, while reducing latency by 16.34% compared to the baseline.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：现代消费级设备（AI PC）已能本地运行大型语言模型（LLM），用户可通过微调或检索增强生成（RAG）个性化自己的边缘LLM智能体，形成多样化的智能体生态系统。
- **核心问题**：单用户的本地智能体专业能力有限（如法律智能体无法处理金融问题），而点对点（P2P）协作可让用户将任务委托给更适合的远程智能体，但面临两大挑战：
  1. **个性化智能体多样性与动态变化（节点流失）**：智能体池不断有加入、离开、更新，且每个智能体专长不同，传统集中式路由无法适应。
  2. **负载不可观测与拥塞**：实时查询到达率变化快，各智能体负载难以被其他用户实时获取，若仅按语义最优路由，会过载部分节点，造成低效。
- **意义**：本文提出PPAI——首个个性化LLM智能体互操作性系统，实现P2P环境下基于智能体专业化的协同边缘智能，显著拓展任务范围并保持负载均衡。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
PPAI由两个主要组件构成：
- **查询-智能体对评分模块（QAGate）**：将查询和所有智能体投影到共享的“原型锚定潜在空间”，通过余弦相似度计算语义兼容性，支持动态变化的智能体池而无需重新训练。
- **多智能体互操作性调度器（贝叶斯博弈）**：每个用户维护对其他智能体负载的信念分布（Belief Distribution），结合语义评分和负载估计，以最大化效用函数（语义相关度减去负载代价），动态选择最优代理。

### 关键技术细节

#### (1) 原型锚定可扩展查询-智能体对评分（§V）
- **架构**：QAGate包含一个轻量级投影器（预训练编码器+两层MLP）和一组可学习的原型向量 {C₁,…,C_K}。
- **训练**：使用带标签查询（聚类得到的20个类别作为软标签）训练投影器，最小化KL散度，使输出分布接近目标分布。
- **推理**：对查询q，计算与每个原型的余弦相似度，经稀疏化处理（保留top-p分数并指数缩放）得到掩码相关向量 ˜f ∈ R^K。
- **智能体能力向量**：每个智能体M_j在同类验证数据上评估，得到其在K个原型上的能力向量 p_j = [p(j,1), …, p(j,K)]（如准确率）。
- **评分**：用户i对查询q与智能体j的匹配度 s(i,j) = ( ˜f^T p_j) / (||˜f||₂ ||p_j||₂)。
- **动态适应**：智能体加入/更新/离开时，通过gossip协议广播能力向量及标志，各节点本地更新，无需全局重训练。

#### (2) 多智能体互操作性贝叶斯博弈（§VI）
- **系统模型**：每个智能体有私有类型 θ_j = (服务率 μ_j, 到达率 λ_j)。用户i维护信念分布 b_i(θ_j)，通过贝叶斯更新从收到的反馈 ω_j 中学习。
- **Cost of Delegation (CoD)**：c(i,j) = 期望队列负载 E[ρ_j] + 推理时间 t_infer + 通信延迟 t_trans（其中 ρ_j = λ_j/μ_j）。
- **效用函数**：U(i,j) = s(i,j) - β·c(i,j)，β>0为权衡超参数。
- **决策**：用户i先通过阈值 θ_s 筛选候选集 E_i，然后请求状态信息更新信念，计算效用，选择 argmax U(i,j)。
- **理论保证**：该博弈是势博弈，存在纯策略贝叶斯纳什均衡；若效用函数严格凹且信念一致，均衡唯一；贝叶斯价格上界 ≤ 5/3；信念收敛到后验。

## 6. 主要结论与发现

- PPAI在多个任务上的**平均准确率最高提升7.96%**（对比最强基线KABB或RouteDC），同时**平均处理延迟降低16.34%**。

- **贝叶斯博弈的信念更新**使得各用户能在负载不可观测时做出接近最优的分布式决策。
