## 摘要
随着基于大语言模型的智能体在边缘-云环境中部署日益广泛，个性化记忆已成为实现长期适应和以用户为中心的交互的关键推动因素。然而，云辅助记忆管理会暴露敏感用户信息，而现有的隐私保护方法通常依赖于激进的掩码策略，这会移除与任务相关的语义，从而降低记忆效用和个性化质量。为解决这一挑战，我们提出MemPrivacy，它在边缘设备上识别隐私敏感跨度，用语义结构化的类型感知占位符替换这些跨度以供云端记忆处理，并在需要时本地恢复原始值。通过将隐私保护与语义破坏解耦，MemPrivacy最小化敏感数据暴露，同时保留有效记忆形成和检索所需的信息。我们还构建了用于系统评估的MemPrivacy-Bench数据集，涵盖200个用户和超过5.2万个隐私实例，并引入了一个四级隐私分类体系以实现可配置的保护策略。实验表明，MemPrivacy在隐私信息提取方面表现出强劲性能，大幅超越了GPT-5.2和Gemini-3.1-Pro等强大的通用模型，同时减少了推理延迟。在多个广泛使用的记忆系统中，MemPrivacy将效用损失限制在1.6%以内，优于基线掩码策略。总体而言，MemPrivacy在隐私保护与个性化记忆效用之间为边缘-云智能体提供了有效的平衡，支持安全、实用且对用户透明的部署。

## Abstract
As LLM-powered agents are increasingly deployed in edge-cloud environments, personalized memory has become a key enabler of long-term adaptation and user-centric interaction. However, cloud-assisted memory management exposes sensitive user information, while existing privacy protection methods typically rely on aggressive masking that removes task-relevant semantics and consequently degrades memory utility and personalization quality. To address this challenge, We propose MemPrivacy, which identifies privacy-sensitive spans on edge devices, replaces them with semantically structured type-aware placeholders for cloud-side memory processing, and restores the original values locally when needed. By decoupling privacy protection from semantic destruction, MemPrivacy minimizes sensitive data exposure while retaining the information required for effective memory formation and retrieval. We also construct MemPrivacy-Bench for systematic evaluation, a dataset covering 200 users and over 52k privacy instances, and introduce a four-level privacy taxonomy for configurable protection policies. Experiments show that MemPrivacy achieves strong performance in privacy information extraction, substantially surpassing strong general-purpose models such as GPT-5.2 and Gemini-3.1-Pro, while also reducing inference latency. Across multiple widely used memory systems, MemPrivacy limits utility loss to within 1.6%, outperforming baseline masking strategies. Overall, MemPrivacy offers an effective balance between privacy protection and personalized memory utility for edge-cloud agents, enabling secure, practical, and user-transparent deployment.

## 2. 方法论

### 核心思想
- **Locally Reversible Pseudonymization**：在边缘侧识别隐私敏感文本跨度，将其替换为包含语义类型的占位符（如 `<Health_Info_1>`、`<Email_1>`），而非简单的掩码符号。映射关系安全存储在本地数据库，云端仅处理脱敏后的输入，最终云端响应中的占位符通过本地查找并恢复为原始值，用户看到完整个性化回复。

### 关键技术细节
1. **四级隐私分类（PL1–PL4）**：
   - **PL1**：低敏感度个性化信息（偏好、习惯），默认不保护。
   - **PL2**：可识别个人信息（姓名、电话、邮箱、详细地址等），可单独或组合识别个体。
   - **PL3**：高敏感度个人信息（健康数据、财务记录、精确位置、生物特征、敏感身份等），泄露可能导致实质性损害。
   - **PL4**：机密/凭证级信息（密码、令牌、API密钥、私钥、恢复码等），可直接用于认证、授权或攻击。
   - 用户可配置保护阈值（如仅保护PL3和PL4），实现灵活隐私-效用控制。

2. **三阶段处理流程**：
   - **阶段1：上行脱敏（Uplink Desensitization）**：本地轻量级MemPrivacy模型检测隐私跨度，输出“原始文本+隐私级别+隐私类型”。根据用户策略，用语义占位符替换保护级别以上的跨度，映射存储于本地安全数据库。非敏感信号和用户授权的信息保持明文。
   - **阶段2：云端处理（Cloud Processing）**：脱敏输入发给云端LLM进行推理和记忆操作。占位符保留语义角色，云端可正常理解任务；泄露的仅是占位符而非真实值。
   - **阶段3：下行恢复（Downlink Restoration）**：云端响应返回后，本地查询数据库，将占位符替换为原值，用户看到流畅个性化回复。恢复过程仅涉及轻量级字符串替换，延迟可忽略。

3. **问题形式化**：定义为约束优化问题，目标最小化隐私泄露风险 \(R_{priv}\)，约束效用损失 \(L_{util} \le \epsilon\)。

4. **模型训练**：
   - **两阶段训练**：
     - **阶段1：监督微调（SFT）**：在26K训练实例上优化标准自回归交叉熵损失，获得基础隐私提取能力。
     - **阶段2：强化学习（GRPO）**：在1K额外实例上使用Group Relative Policy Optimization，奖励函数为提取F1得分，无需额外价值网络，提升准确性和鲁棒性。
   - **模型规模**：基于Qwen3-0.6B、Qwen3-1.7B、Qwen3-4B，训练出6个变体（SFT和RL版本）。

5. **算法流程**（Algorithm 1所示）：
   - 输入原始用户输入X，本地提取器f_θ，保护阈值λ，本地映射数据库D，云端代理C。
   - 检测隐私跨度集P = {(s_i, l_i, t_i)}。

## 6. 主要结论与发现

1. **隐私提取性能卓越**：MemPrivacy模型（0.6B-4B）在MemPrivacy-Bench上F1最高85.97%，在PersonaMem-v2上最高94.48%，显著超越所有通用模型（最佳通用模型F1为78.41%和92.18%）。最小的0.6B模型也优于大多数通用模型。

5. **RL进一步提升性能**：SFT+RL版本比纯SFT版本在隐私提取F1和LLM-as-a-Judge评分上略有提升，证明了偏好优化的有效性。
6. **隐私-效用权衡优越**：随着隐私相关问题比例增加，MemPrivacy的稳定性远超基线方法，尤其在隐私密集场景下优势更明显。
